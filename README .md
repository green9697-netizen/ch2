### 씬 1 (0~2.5초) — "돌봄이 필요한 순간"
- 씬 번호/길이: Scene 01 / 2.5초
- 목표 메시지: 홀로 지내는 어르신의 외로움을 보여주며 공감을 유도한다
- 화면 구성(구도/피사체/배경/텍스트): 미디엄 샷, 아웃포커싱 / 70대 한국인 할머니 / 소박한 한국식 아파트 거실, 작은 식탁, 창문 햇살 / 텍스트 없음
- 내레이션(카피): "누군가의 하루가, 당신의 손길을 기다리고 있습니다." (조용하고 차분한 낮은 톤)
- 사용 도구(목적): 이미지=GPT Image2(감성적 정지 이미지 확보), 비디오=OpenAI Sora 2(약한 줌인으로 정적인 슬픔 강조), 오디오=ElevenLabs(차분한 내레이션)
- 입력 프롬프트(원문): `70대 한국인 할머니가 소박한 한국식 아파트 거실의 작은 식탁에 혼자 앉아 있는 장면. 창문으로 따뜻한 오후 햇살이 들어오고, 테이블 위에는 찻잔이 놓여 있음. 표정은 약간 지치고 외로워 보이지만 전체적으로 감성적이고 따뜻한 분위기. 아웃포커싱 효과, 오렌지·베이지 계열의 따뜻한 색감, 실사 사진 스타일, 고해상도`
- 비디오 변환 프롬프트(모션 지시): `정지된 실내 장면에서 카메라가 아주 천천히 줌인한다. 할머니의 표정과 창가의 햇살은 유지되며 배경은 변경되지 않는다. 정적이고 잔잔한 움직임, 따뜻한 색감 유지.`
- 출력 결과 요약: 따뜻한 오후 햇살 속 홀로 앉은 할머니의 쓸쓸하지만 감성적인 실사 이미지 확보(씬1.png)
- 결과: 최종 영상 0~2.5초 구간

### 씬 2 (2.5~5.5초) — "통합 돌봄 서비스 연결"
- 씬 번호/길이: Scene 02 / 3.0초
- 목표 메시지: 대전시 통합돌봄 서비스가 실제로 가정을 방문해 신뢰를 준다는 것을 보여준다
- 화면 구성: 미디엄 클로즈업 / 30대 한국인 여성 사회복지사 + 어르신 손 / 밝고 깨끗한 실내, 자연광 / 텍스트 없음
- 내레이션(카피): "대전시 통합돌봄이 함께합니다." (밝고 신뢰감 있는 톤)
- 사용 도구(목적): 이미지=GPT Image2(신뢰감 있는 공공서비스 톤앤매너 확보), 비디오=OpenAI Sora 2(살짝 패닝으로 온기 있는 손길 강조), 오디오=ElevenLabs(밝고 또렷한 내레이션)
- 입력 프롬프트(원문, 수정 후): `30대 한국인 여성 사회복지사가 하늘색 조끼 또는 유니폼을 입고 명찰을 달고 있음. 어르신의 가정을 방문하여 태블릿 또는 클립보드를 들고 환하게 웃으며 어르신의 손을 따뜻하게 잡아주는 장면. 밝고 깨끗한 실내, 자연광, 전문적이면서도 따뜻한 분위기, 대전시 공공서비스 홍보 스타일, 실사 사진, 고품질`
- 비디오 변환 프롬프트(모션 지시): `카메라가 사회복지사와 어르신이 맞잡은 손 쪽으로 살짝 패닝한다. 표정은 자연스러운 미소를 유지하며 배경은 변경되지 않는다. 따뜻하고 신뢰감 있는 분위기, 부드러운 카메라 움직임.`
- 출력 결과 요약: 사회복지사가 어르신의 손을 잡는 신뢰감 있는 공공서비스 홍보 스타일 이미지 확보(씬2.png)
- 결과: 최종 영상 2.5~5.5초 구간 

### 씬 3 (5.5~8초) — "행복한 돌봄 결과" (엔딩 씬 · 브랜드 인지 장치 포함)
- 씬 번호/길이: Scene 03 / 2.5초
- 목표 메시지: 돌봄 이후 되찾은 일상의 행복을 보여주고 서비스 신청을 유도한다
- 화면 구성: 와이드샷 / 한국인 노부부 + 중년 돌봄 종사자 / 밝은 거실, 화분·인테리어 소품 / 하단에 "대전시 통합돌봄" 로고 + "지금 신청하세요" CTA 자막(브랜드 인지 장치)
- 내레이션(카피): "일상의 행복을 지켜드립니다. 대전시 통합돌봄 — 지금 신청하세요." (따뜻하고 희망차게, 마지막은 또렷하고 신뢰감 있게)
- 사용 도구(목적): 이미지=GPT Image2(희망적인 와이드샷 확보), 비디오=OpenAI Sora 2(천천히 줌아웃하며 마무리감 연출), 오디오=ElevenLabs(희망적 톤의 클로징 내레이션), 편집=ffmpeg
- 입력 프롬프트(원문): `밝은 거실에서 한국인 노부부와 중년의 돌봄 종사자가 함께 앉아 환하게 웃으며 이야기를 나누는 장면. 테이블 위에는 간식이나 식사가 차려져 있고, 배경에는 화분과 따뜻한 인테리어 소품이 보임. 황금빛 햇살, 희망적이고 훈훈한 분위기, 대전시 지역사회 통합돌봄 캠페인 스타일, 와이드샷, 생동감 있는 따뜻한 색감, 실사 고해상도 사진`
- 비디오 변환 프롬프트(모션 지시): `와이드샷에서 카메라가 천천히 줌아웃하며 장면 전체를 보여준다. 인물들의 웃음과 대화는 자연스럽게 유지되며 배경은 변경되지 않는다. 마무리감 있는 부드러운 움직임, 황금빛 따뜻한 색감 유지.`
- 출력 결과 요약: 노부부와 돌봄 종사자가 웃으며 대화하는 희망적이고 훈훈한 와이드샷 이미지 확보(씬3.png)
- 결과: 최종 영상 5.5~8초 구간(브랜드 장치 구간) 

### 프롬프트 수정 전/후 (씬 2 기준)

**수정 전**: `사회복지사가 할머니 집을 방문하는 장면, 실사 스타일`

**문제점**: 인물의 나이·복장·표정 등 구체 정보가 없어 결과 이미지가 사무적이고 신뢰감이 낮게 나옴(딱딱한 표정, 유니폼 불명확)

**수정 후**: `30대 한국인 여성 사회복지사가 하늘색 조끼 또는 유니폼을 입고 명찰을 달고 있음. 어르신의 가정을 방문하여 태블릿 또는 클립보드를 들고 환하게 웃으며 어르신의 손을 따뜻하게 잡아주는 장면. 밝고 깨끗한 실내, 자연광, 전문적이면서도 따뜻한 분위기, 대전시 공공서비스 홍보 스타일, 실사 사진, 고품질`

| 수정 항목 | 수정 이유 | 결과 변화 |
|----------|----------|----------|
| `30대 여성 사회복지사`, `하늘색 조끼·명찰` 추가 | 인물 나이·복장을 구체화해 공공서비스 담당자임을 명확히 전달 | 사무적 인상이 사라지고 전문성 있는 인물로 개선 |
| `태블릿·클립보드`, `손을 따뜻하게 잡아주는 장면` 추가 | 핵심 행위(방문·연결)를 시각화할 필요 | 방문·돌봄 행위가 명확히 드러나는 구도로 개선 |
| `밝고 깨끗한 실내, 자연광` 추가 | 톤앤매너(신뢰감+따뜻함) 통일 | 밝고 신뢰감 있는 배경으로 개선 |
| `전문적이면서도 따뜻한 분위기` 추가 | 전환 씬의 감정 톤(안심) 명시 | 표정이 자연스러운 미소로 바뀌고 안심 감정이 잘 드러남 |

## 5. 서사 / 메시지 (광고 완성도)

- 구조: **기승전결(문제 제시 → 전환 → 해결/제안)** — 문제(홀로 지내는 어르신의 외로움, 씬1) → 전환(사회복지사의 가정 방문·연결, 씬2) → 해결/브랜드 각인(되찾은 일상의 행복 + CTA, 씬3).
- 마지막 3~5초(씬3, 5.5~8초): 로고 "대전시 통합돌봄" + CTA "지금 신청하세요" 자막 삽입 = 브랜드 인지 장치.
- 톤 아크: 씬1(조용하고 차분한 오렌지·베이지톤) → 씬2(밝고 신뢰감 있는 자연광톤) → 씬3(따뜻하고 희망적인 황금빛톤)으로 점층 전환하여 감정 곡선과 브랜드 각인을 연결.

## 6. 동작 구조 및 설계 (제작 파이프라인)

```
[기획 의도] 브랜드/타겟/메시지 확정(독거 어르신·가족 대상 · 공공 통합돌봄 연결)
  → [프롬프트 설계] 씬별 프롬프트(스타일 레퍼런스 고정: 오렌지·베이지 톤, 실사 사진 스타일)
  → [이미지 생성] GPT Image2 → 키프레임 3종 확정(씬1~3.png, 스토리보드 잠금)
  → [비디오 변환] OpenAI Sora 2 → 정지이미지 기반 카메라 무빙(배경 불변 조건 명시)
  → [오디오] ElevenLabs 내레이션(3카피) + Suno AI BGM
  → [통합 편집] capcut: 컷 연결(2.5+3+2.5) · 로고/CTA 자막 삽입 · 오디오 레벨 조정
  → [출력] daejeon-care.mp4
```

- 설계 의사결정: 이미지 단계에서 인물 나이·표정·배경 톤을 프롬프트로 먼저 고정한 뒤, 씬2에서 발견된 문제(사무적 인상·유니폼 불명확 등)를 프롬프트 수정으로 해결하고 이를 전 씬 스타일 레퍼런스에 반영해 일관성을 확보. 비디오 변환 프롬프트에 "배경은 변경되지 않는다"를 공통 지시어로 명시해 I2V 변환 시 원본 이미지의 인물·배경 정체성을 유지.

## 7. 핵심 기술 원리 (도구별 역할/강점·약점)

- **텍스트→이미지(GPT Image2)**: 디퓨전 기반. 강점=한국식 실내 공간·인물 표정·색감 지시어 반영 정밀도가 높고 감성적인 실사 이미지에 강함. 약점=씬3의 로고·CTA 같은 소형 텍스트 렌더링은 불안정할 수 있어 후합성으로 처리.
- **텍스트/이미지→비디오(OpenAI Sora 2)**: 확산 기반. 강점=정지이미지를 입력으로 배경·인물 정체성을 유지한 채 패닝·줌 등 짧은 카메라 무빙을 안정적으로 생성. 약점=긴 동작(다단계 제스처)에는 불안정 → 씬당 1개의 단순 모션(줌인, 패닝, 줌아웃)으로 프롬프트 제한.
- **텍스트→음성(ElevenLabs)**: 강점=빠른 무저작권 한국어 보이스 생성, 8초 분량 3카피의 감정 톤 변화(안타까움→안심→따뜻함)에 적합. 약점=미세한 감정 조정 한계 → 카피를 짧고 명확하게 구성.
- **일관성 유지**: 스타일 레퍼런스(오렌지·베이지 톤, 실사 사진 스타일) 고정 + 비디오 변환 프롬프트 공통 지시어("배경은 변경되지 않는다") + 톤 아크(차분함→신뢰→희망) 설계로 씬 간 화풍·인물 불일치 방지.

## 8. 심층 인터뷰 대응

- **Q. 이미지/비디오/오디오 도구의 역할과 차이?** → §7. 이미지=정지 키비주얼(GPT Image2), 비디오=카메라 무빙·표정 유지(OpenAI Sora 2), 오디오=내레이션·BGM 레이어(ElevenLabs). 각 단계 출력이 다음 입력이 됨(스틸→I2V, 영상+음성→편집).
- **Q. 같은 요구를 여러 도구로 시도한 결과 차이?** → 씬2 이미지 프롬프트를 수정 전/후로 각각 생성 비교(§2 프롬프트 수정 전/후). 인물 나이·복장·행동 묘사 유무에 따라 결과가 크게 달라짐을 확인, 수정 후 버전을 채택. 
- **Q. 미디어 소스를 해상도/비율/톤 관점으로 어떻게 통합?** → 전 씬 1080p·16:9로 통일(CapCut에서 스케일 조정), 스타일 레퍼런스(오렌지·베이지 톤)로 톤 고정, 차분함→신뢰→희망의 톤 아크로 색 연결. 세로형은 별도 9:16 크롭 출력(보너스).
- **Q. 크레딧/대기열 제약 대응?** → 이미지로 스토리보드를 먼저 확정한 뒤 씬별 I2V 변환은 짧은 단위(2.5~3초)로 분할 생성해 재시도 비용을 최소화. 이미지·비디오·오디오 각각 대체 도구(DALL·E 3 / Pika·Kling / Typecast)를 사전 확보해 접근성 제약에 대응.
- **Q. 60초 버전을 15초로 줄여야 한다면 씬을 어떻게 재구성?** → ① 핵심 메시지 1개(방문·연결 돌봄)만 남기고 감정 고조 컷을 축약. ② 씬당 5초 내 단축, 전환 컷 위주로 재편. ③ 내레이션은 핵심 카피 1줄 + 엔드카드 CTA만 유지. ④ 3씬 구조를 문제(1)/해결(1)/브랜드(1)로 유지하되 각 씬 길이만 확장. ⑤ 브랜드 장치(로고·CTA)는 마지막 3~5초 필수 보존.

## 제약/환경 기록

- 도구·채널·설정: 이미지=GPT Image2(웹) / 비디오=OpenAI Sora 2 / 음성·BGM=ElevenLabs / 편집=capcut.
